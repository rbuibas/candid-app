# Manual exploration testing

An account of manual testing — not following any test steps, just trying the app out and seeing how it behaves.

Below is a running list of bugs or feature requests with explanations. For each one, a GitHub issue
gets created, then the bug is removed from this document so it stays a clean
scratchpad for the next round of testing. Issue descriptions carry the shared
info, curated, plus any quick hints — no in-depth investigation up front.

If something is not a bug, but a new feature, make sure to signal that in the title of the issue.

If it's a bug mark as bug, if feature mark as feature.
